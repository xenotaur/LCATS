# Guardrails: Safety

## Safety Constraints
- Prefer additive, reversible project-management changes over implementation risk during LRH control-plane evolution.
- Do not represent speculative architecture as confirmed system behavior.
- Keep data-source and model-usage claims aligned with documented repository behavior.

## Validation Posture
- Status claims should cite evidence artifacts and observable repository signals.
- Unknowns should remain explicit until validated.
