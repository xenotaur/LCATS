# Guardrails: Approvals

## Approval Policy (Bootstrap)
- Changes to authoritative artifacts should be reviewed before being treated as operational commitments.
- Derived context files should be regenerated or updated when authoritative artifacts change materially.

## High-Impact Changes Requiring Explicit Approval
- Goal/scope changes in `goal/project_goal.md`.
- Health/status color changes in `status/current_status.md` without new evidence entries.
- Any proposal to treat uncertain roadmap items as committed milestones.
