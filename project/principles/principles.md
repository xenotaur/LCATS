# Principles

1. **Evidence before assertion**: Claims in control-plane artifacts should be grounded in repository evidence (README/docs/path-level signals) or marked uncertain.
2. **Non-destructive execution**: LRH artifacts should be additive and not alter existing implementation behavior.
3. **Conservative interpretation**: When signals conflict or are incomplete, preserve uncertainty and avoid speculative commitments.
4. **Precedence discipline**: Interpret and execute using `principles → goal → roadmap → focus → work_items → evidence → status`.
5. **Auditable progress**: Status claims should be traceable to evidence entries and decision log rationale.
