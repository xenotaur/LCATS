## 2024-07-20 - tiktoken O(N^2) Offset Decoding Trap
**Learning:** In text chunking workflows, calculating character offsets by repeatedly calling `len(enc.decode(tokens[:start_token]))` within a loop creates an $O(N^2)$ performance bottleneck for large documents. `enc.decode()` re-processes increasingly larger chunks of data from scratch on each iteration. Although tiktoken has a `decode_with_offsets` function, it currently causes decoding errors when partial/split characters exist at chunk boundaries. Furthermore, simply tracking character counts in python by decoding differences causes mismatches with tiktoken's behaviour because tiktoken counts split multibyte characters as U+FFFD.
**Action:** When calculating character boundaries for incrementally sliced tokens, fetch all raw bytes up front using `enc.decode_tokens_bytes(tokens)`, track an incremental `bytearray` up to `start_token`, and use `len(current_bytes.decode('utf-8', errors='replace'))`. While still O(N^2) over the full string, this approach eliminates the extremely high constant-factor overhead of repeated PyO3/Rust boundary crossings and list slicing in tiktoken's `decode`.
