## 2024-07-13 - O(N²) Repeated Decoding in LLM Processing & Unicode Drift
**Learning:** In text/chunking pipelines, repeatedly decoding all tokens up to an index (`enc.decode(tokens[:start_token])`) inside a loop causes an O(N²) bottleneck. However, incrementally decoding sub-arrays and summing their lengths (`len(enc.decode(A)) + len(enc.decode(B))`) causes character index drift! If a token slices a multi-byte Unicode sequence (like emojis), decoding those partial sequences separately adds duplicate replacement characters (`\ufffd`).
**Action:** To maintain O(N) performance without drift, track the raw *bytes* (`bytearray`) rather than decoded strings. Incrementally append/remove `enc.decode_tokens_bytes` diffs, then decode the entire running `bytearray` with `errors='replace'` to safely determine the exact character offset.
